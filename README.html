Projeto Login Completo

Este projeto é composto por uma API de login e uma interface de login. A API de login foi desenvolvida em Java utilizando o framework Spring Boot e gerenciamento de dependências Maven. A interface de login foi desenvolvida em JavaScript usando o framework React e npm para gerenciamento de dependências.

Funcionalidades

  • Autenticação de usuários
  • Criação de novos usuários
  • Validação de CPF (API)
  • Geração de ID de usuário como hash (API)
  • Validação de email (Interface)
  • Validação de senha (Interface)

Estrutura do Projeto

O projeto segue a estrutura padrão de um projeto Spring Boot com Maven para a API e a estrutura padrão de um aplicativo React para a interface. Aqui estão algumas das classes principais e suas responsabilidades:

  • UserController: Controlador REST para manipulação de usuários (API).
  • AuthController: Controlador REST para autenticação de usuários (API).
  • UserService: Serviço para operações relacionadas ao usuário (API).
  • UserRepository: Repositório para interação com o banco de dados de usuários (API).
  • CpfValidator: Classe utilitária para validação de CPF (API).
  • src/components: Diretório com componentes individuais (Interface).
  • src/index.js: Ponto de entrada do aplicativo (Interface).

Tecnologias Utilizadas

  • Java (API)
  • Spring Boot (API)
  • Maven (API)
  • H2 Database (API)
  • JavaScript (Interface)
  • React (Interface)
  • npm (Interface)

Como Executar

Para executar este projeto, você precisa ter o Java e o Maven instalados em seu sistema para a API, e Node.js e npm para a interface.

  1. Faça o clone do repositório para o seu sistema local.
  2. Navegue até o diretório do projeto.
  3. Para a API, execute o comando mvn spring-boot:run para iniciar a aplicação. A aplicação estará disponível no endereço http://localhost:8080.
  4. Para a interface, execute o comando npm install para instalar as dependências. Em seguida, execute o comando npm start para iniciar a aplicação. A aplicação estará disponível em http://localhost:3000.

Endpoints

Os seguintes endpoints estão disponíveis neste projeto:

Auth Controller

  • PUT /auth/recovery-password: Recupera a senha de um usuário.
  • POST /auth/signup: Cria um novo usuário.
  • POST /auth/authenticate: Autentica um usuário existente.

User Controller

  • PUT /api/users/change-role/{email}/{newRole}: Altera a função de um usuário.
  • GET /api/users: Retorna todos os usuários.
  • GET /api/users/{email}: Retorna um usuário específico com base no email.
  • DELETE /api/users/{email}: Exclui um usuário com base no email.
  • GET /api/users/me: Retorna o usuário atualmente autenticado.

Public Controller

  • GET /public/numberOfUsers: Retorna o número total de usuários.
